# 📖 从 0 到 1：Lumière AI 美妆导购引擎架构日记

## 💡 1. 业务痛点洞察：传统客服体系的“三高一低”困境
在深度参与电商美妆的高频咨询场景后，我发现传统的“关键词客服机器人”与“纯人工接待”模式已全面失效，面临极其致命的摩擦力：
* **语义鸿沟（听不懂）**：消费者习惯使用复杂的长句或“绝绝紫”、“早泥豆沙”等民间黑话。传统机器人依赖正则匹配，极易导致转人工或跳失。
* **策略脱节（瞎推荐）**：纯大模型客服虽然能聊，但缺乏全局商业视野，极易产生商品幻觉，将“已断货”商品推给客户引发客诉，或无法执行品牌强制曝光新品的战略。
* **物料爆炸（改不动）**：任意两款色号的对比组合数呈指数级增长。遇到换季上新，客服训练师需要手动修改上百条知识库话术，并依赖设计团队出图，工作量极其繁琐重复。

## 🧠 2. 核心破局点：防幻觉“双轨制”三车间架构
为了彻底解决上述痛点，我放弃了常规的大模型直连方案，独创了 **“LLM（意图解码） + Pandas（规则引擎） + Python（视觉工坊）”** 的三车间混合架构。

* **第一车间：AI 意图解码大脑（解决“听懂”问题）**
    挂载《产品花名册》别名字典，只让大模型干它最擅长的事：降噪与意图对齐。将消费者的情绪废话和野生黑话，精准映射并输出为干净的结构化 JSON（产品词 + 场景标签）。
* **第二车间：核心业务推荐漏斗（解决“库存与策略”问题）**
    这是系统的“白盒中枢”，纯代码执行，零大模型介入。接收标签后，通过 Pandas 漏斗执行绝对物理隔离（一票否决已断货商品）。随后进行多维算分，并硬编码执行系列战略优先级与“老带新”强制推新策略。确保推给用户的商品 100% 在售且契合商业利益。
* **第三车间：情绪包装与视觉工坊（解决“转化”问题）**
    将第二车间选出的冰冷 SKU 与一字不差的官方档案重新喂给 LLM，结合用户语境（如：约会、通勤）生成高情商逼单话术。

## 🎨 3. 降维打击：自动化视觉营销引擎
面对海量的色号对比需求，系统底层的 `auto_stitch_v2.py` 图像引擎接管了传统的“人工修图”工作。
* **自适应拼图**：利用 Pillow 库自动读取高清原图，执行动态尺寸计算、高级留白与排版。
* **毫秒级渲染**：自动提取标准英文色号，合成带双色号品名与品牌水印的对比海报。实现了千人千图的“所见即所得”，极大地缩短了消费者的决策链路。

## 📈 4. 数据飞轮：B 端大盘沉淀与 RLHF 反哺
技术最终需要反哺业务。本系统并不是聊完即走的工具，而是品牌的数字资产沉淀器：
* **隐形埋点与商业洞察**：C 端交互瞬间，底层静默写入日志。在 B 端实时可视化“高频野生黑话词云”与“漏斗转化率”，直接指导电商店铺的 SEO 优化与供应链备货。
* **RLHF 标注大盘**：构建了人工复盘标注台，形成“C 端交互 -> 数据入库 -> B 端专家打分”的数据飞轮，为后续微调品牌专属私有大模型积累高质量数据对。

## 🗺️ 5. 技术演进路线图 (Future Roadmap)
作为一款具备 SaaS 雏形的引擎，其未来演进规划如下：
1.  **$O(1)$ 边际扩展架构**：彻底解耦知识库，实现原子化特征组合。扩充千款 SKU 的边际话术产出成本为零，瞬间完成跨品类泛化。
2.  **RAG 向量检索接入**：引入 ChromaDB/Milvus 向量库，对品牌数十万字的《产品培训手册》进行 Embedding，实现长尾专业知识的零幻觉问答。
3.  **Agentic 工作流重构**：引入 LangGraph 状态机，将线性链路升级为具备循环推理与自我纠错能力的多智能体（Multi-Agent）系统。
4.  **多模态视觉试妆（Vision-Language）**：接入视觉大模型，支持消费者上传网图或自拍，系统自动提取肤色与色彩 RGB，进行跨模态对比推荐。
